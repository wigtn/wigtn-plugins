# Page State Checklist

화면 1개당 5가지 상태를 검토한다. PRD §5.4.1에서 체크된 항목은 03-SCREEN-SPEC.md에서 1줄 이상 명세 필수.

## 1. loading

데이터를 가져오는 중. 사용자가 빈 화면을 보지 않도록.

| 패턴 | 사용 시점 |
|------|----------|
| Skeleton screen | 리스트/카드. 실제 레이아웃을 회색 박스로 미리 그림 |
| Spinner | 버튼 disabled + 가운데 스피너. 폼 제출 등 짧은 작업 |
| Progress bar | 파일 업로드, 다운로드 |
| Streaming | LLM 응답, 채팅 메시지 |

**체크**:
- [ ] 스피너만 단독으로 큰 화면에 띄우지 않는다 (skeleton 우선)
- [ ] 1초 미만이면 표시하지 않는다 (깜빡임 방지)
- [ ] 버튼 loading 중에는 disabled + 라벨 변경 (예: "제출 중...")

## 2. empty

정상 응답이지만 결과가 0건. 가장 자주 빠뜨리는 상태.

| 패턴 | 메시지 예시 |
|------|------------|
| 첫 사용자 | "첫 골든셋을 작성해보세요" + CTA |
| 필터 결과 없음 | "조건에 맞는 항목이 없습니다" + 필터 초기화 |
| 검색 결과 없음 | "'{query}'에 대한 결과가 없습니다" + 다른 검색 제안 |

**체크**:
- [ ] CTA(다음 행동 유도) 있는가
- [ ] 빈 상태 일러스트나 아이콘 (선택)
- [ ] "데이터 없음" 같은 무미건조한 표현 지양

## 3. error

4xx/5xx 응답 또는 클라이언트 검증 실패.

| 에러 종류 | 처리 |
|----------|------|
| 4xx 클라이언트 (검증) | 필드 옆 inline 메시지 |
| 422 비즈니스 규칙 위반 | 상단 배너 + 영향 필드 하이라이트 |
| 401 인증 만료 | 자동 로그인 모달 또는 / 리다이렉트 |
| 403 권한 없음 | no-permission 페이지로 전환 |
| 5xx 서버 | "잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 네트워크 실패 | offline indicator + 재연결 시도 |

**체크**:
- [ ] 에러 메시지가 사용자 언어인가 ("ENOTFOUND" 같은 코드 노출 금지)
- [ ] 다음 행동이 명확한가 (재시도 / 다시 입력 / 문의)
- [ ] 재시도 횟수가 무한이 아닌가 (지수 백오프)

## 4. success

정상 응답 + 결과 ≥1건. 기본 상태.

| 추가 처리 | 시점 |
|----------|------|
| Toast 알림 | mutation 성공 후 ("저장되었습니다") |
| 페이지 전환 | 폼 제출 후 결과 페이지로 |
| Optimistic update | 좋아요/북마크처럼 즉시 반영해도 안전한 작업 |

**체크**:
- [ ] 성공을 시각적으로 인지할 수 있는가 (배지/색/애니메이션)
- [ ] Optimistic update 실패 시 롤백 처리

## 5. no-permission

인증은 됐으나 권한 부족. RBAC 페이지에서 필수.

| 처리 | 시점 |
|------|------|
| 메뉴 자체 숨김 | admin 메뉴를 일반 사용자에게 표시하지 않음 |
| 페이지 접근 시 리다이렉트 | URL 직접 입력한 경우 / 로 |
| 안내 페이지 | "관리자 권한이 필요합니다. PM에게 문의하세요" |

**체크**:
- [ ] 클라이언트 가드만 두지 않는다 (서버 RLS 필수)
- [ ] 권한 부족이 일반 에러처럼 보이지 않는다 (별도 메시지)

## State Coverage Matrix Template

화면 명세 작성 시 표로 정리:

```markdown
### States
- [x] loading: <구체적 처리>
- [ ] empty: N/A
- [x] error: <구체적 처리>
- [x] success: <구체적 처리>
- [ ] no-permission: N/A
```

체크박스가 모두 비어 있으면 → 상태가 정의되지 않은 화면이므로 재검토.
