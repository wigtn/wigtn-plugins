# Microcopy Patterns

화면정의서 작성 시 자주 빠지는 마이크로카피. 사용자가 화면에서 만나는 모든 문자.

## 1. 버튼 라벨

| 안티 | 권장 | 이유 |
|------|------|------|
| "확인" | "저장하기" / "삭제하기" / "전송하기" | 동작이 명확 |
| "OK" | (액션 동사) | 영문 OK는 한국어 화면에 부자연 |
| "Submit" | "제출하기" | 일관된 한국어 |
| "Cancel" | "취소" | 익숙한 표현 |

**규칙**:
- 명사가 아니라 **동사** ("저장" → "저장하기")
- 라벨 길이 ≤ 6자 권장
- 1차 액션은 강조 (primary), 2차는 약화 (ghost/text)

## 2. 빈 상태 메시지

| 상황 | 안티 | 권장 |
|------|------|------|
| 첫 사용자 | "데이터 없음" | "첫 항목을 만들어보세요" + CTA |
| 검색 0건 | "결과 없음" | "'{query}'에 대한 결과가 없습니다. 다른 단어로 검색해보세요" |
| 필터 0건 | "비어있음" | "조건에 맞는 항목이 없습니다" + "필터 초기화" |

## 3. 에러 메시지

### 폼 검증 (필드별)

```
[안티] "Invalid input"
[권장] "이메일 형식이 올바르지 않습니다 (예: user@example.com)"

[안티] "Required"
[권장] "필수 입력 항목입니다"

[안티] "Too short"
[권장] "최소 10자 이상 입력해주세요 (현재 5자)"
```

### 서버 에러

```
[안티] "ENOTFOUND" / "500 Internal Server Error"
[권장] "잠시 후 다시 시도해주세요. 문제가 계속되면 문의 부탁드립니다."

[안티] "Unauthorized"
[권장] "로그인이 만료되었습니다. 다시 로그인해주세요."
```

### 비즈니스 규칙 위반

```
[안티] "VALIDATION_FAILED"
[권장] "민감 정보(이메일/내부 URL/API 키)가 포함된 것 같습니다. 확인 후 다시 제출하세요."
```

## 4. 진입 안내 / 도움말

폼 상단, 페이지 헤더 아래에 1~2줄.

```
[좋음] "작성에 평균 10분 소요됩니다. 임시저장 가능합니다."
[좋음] "FAQ는 월 5건 이상 반복되는 질문을 의미합니다."
[좋음] "100% 완벽한 정답일 필요는 없습니다. W6 채점 룰 합의 라운드에서 다시 검토됩니다."
```

**규칙**:
- 사용자의 부담을 덜어주는 톤
- "왜 이걸 묻는지" 명확히

## 5. 확인 모달 (Confirmation)

```
[안티] "정말 삭제하시겠습니까?"
       [확인] [취소]

[권장] "이 항목을 삭제합니다"
       "삭제된 항목은 복구할 수 없습니다."
       [삭제하기] [취소]
```

**규칙**:
- 제목: 명확한 동작
- 본문: 결과/영향 1줄
- 버튼: 동작이 라벨로 그대로 (확인 X)
- 위험한 액션은 빨간색 (destructive)

## 6. Toast / Snackbar

성공/실패 알림. 짧고 명확하게.

```
[성공] "저장되었습니다"           — 2초 자동 사라짐
[성공] "제출되었습니다. 검토 후 알려드립니다."
[정보] "임시저장 중..."
[경고] "변경사항을 저장하지 않고 나갑니다"   — 사용자 확인 필요
[에러] "저장 실패. 네트워크를 확인하세요"     — 사용자가 닫을 때까지
```

## 7. 상태 배지 (Status Badge)

| status | 라벨 | 색상 (Tailwind) |
|--------|------|---------------|
| draft | 작성중 | `bg-neutral-100 text-neutral-700` |
| submitted | 제출됨 | `bg-blue-100 text-blue-700` |
| reviewed | 검토중 | `bg-amber-100 text-amber-700` |
| approved | 승인됨 | `bg-green-100 text-green-700` |
| rejected | 반려됨 | `bg-red-100 text-red-700` |

## 8. 권한 안내

```
[안티] "Access Denied"
[권장] "관리자 권한이 필요합니다. 권한이 필요하면 {담당자 또는 팀}에게 문의하세요."

[안티] "Forbidden"
[권장] "이 항목은 작성자만 수정할 수 있습니다."
```

## 9. 작성 가이드 / 인라인 도움말

폼 필드 옆 또는 placeholder.

```
[좋음] placeholder="예: 시공 중 캐스터로 보강 방법 알려줘"
[좋음] "쉼표 또는 엔터로 구분: 4, 15, 41"
[좋음] "현장 어휘를 그대로 사용하세요"
```

## 10. AI 냄새 피하기

다음 표현은 자주 클리셰로 인식됨:

| 안티 (AI 냄새) | 권장 |
|--------------|------|
| "쉽고 빠르게" | (구체적 시간/단계) |
| "스마트한 솔루션" | (구체적 기능) |
| "혁신적인" | (실제 차별점) |
| "당신의 모든 것을" | (실제 범위) |

**규칙**: 형용사보다 **숫자, 단계, 예시**로 표현.
